# p1nano-host — моторный фейдер iCON P1-Nano → громкость macOS

Превращает свободный DAW-слот контроллера **iCON P1-Nano** в полноценный **двусторонний
регулятор системной громкости macOS**: ведёшь моторный фейдер — меняется громкость мака
(например, смотришь YouTube), меняешь громкость с клавиатуры — моторный фейдер сам едет.
Плюс системная HUD-плашка громкости на экране и кнопка Play/Pause.

Хост по протоколу **Mackie Control Universal (MCU)** «прикидывается DAW», будит устройство
на выбранном порту и работает с ним напрямую через CoreMIDI. Ableton при этом не нужен —
слот просто должен быть в режиме, дающем фейдеру pitch-bend (Ableton Live / Mackie Control).

## Возможности

- **Фейдер → громкость.** Ведёшь фейдер → громкость мака меняется плавно, на полном
  разрешении (CoreAudio), верх фейдера = 100 %.
- **Громкость → фейдер.** Меняешь громкость с клавиатуры/мышью → моторный фейдер сам
  едет к новому значению.
- **HUD-плашка.** На экране появляется системная плашка громкости — как при нажатии
  клавиш громкости на клавиатуре. Через приватный `OSDManager` из `OSD.framework`.
- **Play/Pause.** Кнопка транспорта → медиа-клавиша Play/Pause macOS.
- **Без клацанья мотора.** Мотор получает позицию только при реальном изменении громкости
  и точно удерживается на отпускании; в покое молчит.

## Запуск

Переключи P1-Nano на свободный DAW-слот (у меня **DAW 3 = Ableton Live** → даёт фейдеру
pitch-bend на «Порт 3»). Затем:

```bash
bash start.sh
```

Хост можно запускать даже до включения iCON — он ждёт устройство и подключается сам.
Остановить: `Ctrl+C`.

Запуск вручную (с подробным логом):

```bash
DBG=1 ./host "Порт 3"      # на железе
DBG=1 ./host "P1Sim"       # против локального симулятора
```

## Как это устроено

### Протокол / «пробуждение» устройства
- Хост шлёт устройству непрерывный MCU-поток: метеры (`F0 00 00 66 14 20 …`),
  запрос версии прошивки (`14 13`), текст на LCD (`14 12`). Это «будит» слот — на экране
  появляется «Mac Volume», и фейдер начинает слать свою позицию.
- Физический фейдер шлёт **Pitch Wheel на канале 1**; касание фейдера = нота **104 (G♯6)**.
- Кнопки транспорта в режиме Ableton шлют ноты **93 (Stop) / 94 (Play)** (в MIDI-мониторе
  отображаются как `A5` / `A♯5`).

### Громкость
- Ставится мгновенно через **CoreAudio** (`kAudioDevicePropertyVolumeScalar`), без
  запуска `osascript` — поэтому плавно и без лагов.

### HUD-плашка
- `OSDManager.showImage:onDisplayID:priority:msecUntilFade:filledChiclets:totalChiclets:locked:`
  (image = 3 — динамик, 16 «чиклетов»). Вызов через IMP метода (типы `q,I,I,I,I,I,B`).
  Работает на macOS 26 (Tahoe).

### Play/Pause
- `NSEvent.otherEvent(.systemDefined)` с `NX_KEYTYPE_PLAY = 16` → `CGEvent` на
  `.cghidEventTap`.

### Архитектура потоков (ключевое — из-за этого раньше падало)
Первая версия дёргала CoreAudio/AppKit/OSDManager **прямо из MIDI-callback’а**. AppKit не
потокобезопасен — при быстром потоке pitch это роняло процесс («работает, потом умирает»).
Текущая версия разделена:

- **MIDI-поток** (CoreMIDI callback) — только парсит байты настоящим потоковым парсером
  (с поддержкой **running-status**: быстрый фейдер слипает несколько pitch-сообщений в один
  пакет) и кладёт состояние под `NSLock`. Никаких тяжёлых вызовов отсюда.
- **Главный поток**, таймер **30 Гц** — забирает последнее состояние и делает всю работу:
  ставит громкость, рисует HUD, шлёт Play/Pause, удерживает мотор. Берётся только последнее
  значение фейдера → плавно, без флуда.
- Отдельные таймеры: обратная связь громкость→фейдер (10 Гц) и keepalive/online (4 Гц).

## Файлы

| Файл | Назначение |
|---|---|
| `host.swift` | Хост (главный). Сборка: `swiftc host.swift -o host`. |
| `devicesim.swift` | Локальный симулятор P1-Nano (виртуальный порт `P1Sim`) — тесты без железа. |
| `start.sh` | Запуск на железе (Порт 3). |
| `test.sh` | Тест «фейдер → громкость» (через симулятор). |
| `fbtest.sh` | Тест «громкость → фейдер» (обратная связь). |

Бинарники `host` / `devicesim` не коммитятся — собираются из `.swift`.

## Локальные тесты (без железа)

```bash
bash test.sh     # фейдер -> громкость  (ожидается диапазон ~7..92)
bash fbtest.sh   # громкость -> фейдер  (мотор едет на 20/80/35)
```

## Настройка

- **Порт.** Аргумент хоста — подстрока имени MIDI-порта (`"Порт 3"`). Совпадает с DAW-слотом.
- **Кнопка Play.** `PLAY_NOTES` в `host.swift` (по умолчанию `[91,92,93,94,95]` — транспорт).
- На DAW 1 (Ableton с MackieControlFollow) хост одновременно не запускать — это другой слот,
  чтобы не делить физический фейдер.

## Зависимости

macOS, Swift (Xcode CLT). Только системные фреймворки: CoreMIDI, CoreAudio, AppKit,
приватный OSD.framework. Внешних пакетов нет.
